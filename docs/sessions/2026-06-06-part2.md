## Key decisions

1. **HTML+SVG over Phaser, second confirmation.** v121 Cauldron rebuild + v126 Explorer's Hall rebuild both dropped Phaser. Pattern: any new or rebuilt zone uses HTML+SVG+CSS unless arcade physics are essential.
2. **Pedagogy over mechanics.** v125 rewrote the Cauldron play loop to maximize cognitive moments per session — bunch counting (subitization), missing addends, make-ten strategy, ten-frame visualization. Same principle drove v126 Explorer's Hall design.
3. **Embedded pulsating Humphrey tab as a reusable pattern.** First built in v125, refined in v127. Any zone with active prompts now embeds a 64px Humphrey portrait that pulses on new utterance. Global floater hidden. All TTS routed direct through `/api/humphrey/tts`.
4. **`audioEnabled: false` on global Humphrey for new zones.** Avoids the welcome-event-falls-to-default trap (root cause of both v123 and v127 bugs). Documented as a hard rule in HANDOFF.md.
5. **Old zone files stay in repo on deprecation.** `diner-lanes.html` left intact + still in SW CORE despite router pointing elsewhere. Enables clean rollback.

## The Humphrey audio anti-pattern (struck twice this session)

The bug shipped silently in v121 and didn't surface until v123 (user reported "she keeps making the same sound"). Same root-cause shape appeared again in v126 and was caught proactively during live UI verification.

**Pattern:**
- `humphrey.say()` expects `(event: string, context: object)`
- If `event` is not a string OR is a string not registered in CATALOG, the function falls through to `CATALOG._default` which has 4 generic filler lines
- The user hears generic fillers instead of intended dynamic text
- Audio overlay also a risk: custom intro from embedded tab + fallback default from global Humphrey can play simultaneously

**v123 fix scope:** all 9 Cauldron `safeSay` call sites updated to correct signature with specific event IDs.

**v127 fix scope:** Explorer's Hall global Humphrey init updated to `audioEnabled: false` so the unregistered welcome event can't auto-play.

**Rule going forward:** for any zone using the embedded Humphrey tab pattern, ALWAYS set `audioEnabled: false` on the global init.

## Cooking Math pedagogical design (v125)

The most substantial pedagogical shift this session. Old Cauldron loop: tap a veggie card 9 times, badge counts up to 9, repeat for tomatoes and potatoes, type the sum. Zero math actually happens — the game does the counting AND the addition.

New loop per ingredient:
1. **Bunch arrives in basket** (e.g. 5 carrots, visually grouped)
2. **"How many in this bunch?"** — 3-button quick tap, correct + ±1 distractors
3. **Carrots fly into cauldron ten-frame** — visual filling, slot-by-slot
4. **Missing-addend bridge:** "We have 5, recipe needs 9, how many more?" — 3-button
5. **Next bunch arrives with that count** — fly in
6. **Cross-ingredient bridge:** "9 carrots in pot. Need 8 tomatoes. After adding, how many total?"

Bunch sizes chosen for pedagogical intent:
- 6 → [3,3] (doubles)
- 8 → [4,4] (doubles)
- 9 → [5,4] (make-ten setup)
- 11-15 → [5,5,N] (10+N place value)
- 16-20 → [5,5,5,N] (extra make-ten step)

Ten-frame inside cauldron visualizes counts directly. Multi-row for teens to show 10+N place value.

Per recipe: ~11 cognitive moments. Per session (5 recipes): ~55 mental-math reps. Old loop: 0.

## Explorer's Hall expedition design (v126)

Diner Lanes was a Phaser bowling game where 8 hardcoded customers from US states ordered their state's stereotypical food. The "social studies" was reduced to state→food trivia, no real history, civics, geography, or culture.

New loop per visit:
1. **Discovery card** — illustration + 2-3 sentence intro that Humphrey reads (Annapolis State House for v126)
2. **3 wonder tiles** (purple/emerald/amber) — tap to reveal fact + question
3. **Quick-check question** per wonder — 3 answer buttons, wrong path shows hint + brass-highlighted correct
4. **Passport stamp animation** on completing all 3 wonders
5. **Topbar passport count** persists across sessions via localStorage

v126 ships Phases 1, 2, 3, 6 with hardcoded Annapolis content. v128 swaps to Haiku-generated daily expedition; v129 adds Supabase persistence and home-screen passport tile; v129/v130 adds Phases 4-5.

Topic rotation when v128 lands: Maryland history (30%), US history (25%), geography (15%), civics (15%), cultures (15%). Profile-personalized.
