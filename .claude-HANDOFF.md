### Supabase patterns (v140 reinforced)
- All writes via `SECURITY DEFINER` RPCs with explicit `GRANT EXECUTE TO anon, authenticated, service_role`
- Generic `ha_record_event(child_id, event_type, payload)` for telemetry
- `DROP FUNCTION IF EXISTS` + `CREATE FUNCTION` when return type changes (used in v140 for `ha_get_today_voice_usage`)
- Mastery threshold: ≥80% in session
- Daily voice cap: 15 conversations/day via `ha_get_today_voice_usage` RPC; event_type pattern `%_voice` aggregates across all zones

### ConvAI patterns (v140 new)
- Client tools registered in dashboard with JSON schemas → client provides implementations via `clientTools: { name: fn }` in `Conversation.startSession()`
- Tools accept a single `params` object — destructure named fields with defaults
- Tools return a string (becomes the tool result Humphrey sees)
- `conversation.sendContextualUpdate(text)` injects context without triggering a response — ideal for ambient awareness updates like canvas vision
- Dynamic variables passed at session start: `dynamicVariables: { var_name: value }` — these match `{{var_name}}` references in the dashboard prompt
